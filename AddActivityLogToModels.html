Activity Log Trait Installer Command

A Laravel Artisan command that automatically adds Spatie's Activity Log trait to Laravel models.

Command Overview

php artisan models:add-activity-log [options] [models]

Description

This command automates the process of adding activity logging capabilities to Laravel models by:

  • Adding the LogsActivity trait
  • Implementing the LogOptions interface
  • Adding the getActivitylogOptions() method
  • Managing file backups
  • Providing safe dry-run capabilities

Arguments

Argument Description
models Optional. Specific model names to process (without .php extension)

Options

Option Description Default
--dry-run Show what would be changed without making changes false
--backup Create backups before modifying files false
--restore Restore from the most recent backup false
--restore-from Restore from a specific backup timestamp null
--keep-backups Number of backups to keep 5

Usage Examples


# Add activity log to all models
php artisan models:add-activity-log

# Add activity log to specific models
php artisan models:add-activity-log User Post Comment

# Preview changes without modifying files
php artisan models:add-activity-log --dry-run

# Add activity log with backup
php artisan models:add-activity-log --backup

# Restore from most recent backup
php artisan models:add-activity-log --restore

# Restore from specific backup
php artisan models:add-activity-log --restore-from=2024-03-20_123456

# Limit backup history
php artisan models:add-activity-log --backup --keep-backups=3

Features

Model Modifications

The command adds the following to your models:

1. Required Use Statements

use Spatie\Activitylog\Traits\LogsActivity;
use Spatie\Activitylog\LogOptions;

2. Trait Implementation

use LogsActivity;

3. Activity Log Configuration Method

public function getActivitylogOptions(): LogOptions
{
    return LogOptions::defaults()
        ->logOnly(['*'])
        ->logOnlyDirty();
}

Code Analysis and Modification

  • Uses PHP Parser for safe code modification
  • Maintains existing code formatting
  • Preserves comments and docblocks
  • Handles different trait declaration styles
  • Adds proper spacing and line breaks

Backup Management

  • Creates timestamped backups in storage/app/backups/models/YYYY-MM-DD_HHMMSS/
  • Maintains configurable number of backup versions
  • Automatic cleanup of old backups
  • Interactive backup restoration
  • Backup rotation with configurable limit

Safety Features

1. Input Validation

  • Validates model names (alphanumeric and underscores only)
  • Prevents directory traversal attempts
  • Validates backup timestamps
  • Validates numeric inputs

2. Dry Run Mode

  • Shows detailed diff of proposed changes
  • Color-coded output for better readability
  • No files are modified
  • Safe for testing in production

3. Backup System

  • Automatic backups before modifications
  • Multiple backup versions
  • Easy restoration process
  • Configurable backup retention

Error Handling

The command handles various error scenarios:

  • Invalid model names
  • Non-existent models
  • File permission issues
  • Parse errors in PHP files
  • Backup/restore issues
  • Directory access problems

Requirements

  • Laravel 8.0+
  • Spatie Laravel Activity Log package installed
  • PHP 8.0+
  • Write permissions on models directory
  • Write permissions on backup directory

Best Practices

  1. Always use --dry-run first to preview changes
  2. Create backups with --backup when modifying files
  3. Keep reasonable number of backups (3-5 recommended)
  4. Test on development before running on production
  5. Verify changes after application
Note: The command uses PHP Parser to safely modify your code while preserving formatting and comments.
Warning: Always backup your files before running this command in production.
